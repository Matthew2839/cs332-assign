1) 최초 세팅 / 사용자 정보
git config --global user.name  "Your Name"
git config --global user.email "your_email@example.com"
git config --global -l      # 글로벌 설정 확인

2) 새 레포 만들기 & 원격 연결
cd D:\path\to\project
git init
git branch -M main
git remote add origin https://github.com/<id>/<repo>.git
git remote -v                # 원격 확인

3) 일상 루틴 (상태→스테이징→커밋→푸시)
git status                   # 변경사항 확인
git add -A                   # 모든 변경 스테이징
git commit -m "메시지"        # 커밋 생성
git push -u origin main      # 최초 1회 -u로 업스트림 설정
git push                     # 이후엔 이거만

4) 브랜치 기본
git branch                   # 브랜치 목록/현재표시(*)
git checkout -b feature/x    # 새 브랜치 생성+이동
git checkout main            # 브랜치 이동
git merge feature/x          # 현재 브랜치로 병합
git branch -M main           # 기본 브랜치 이름 변경(master→main)

5) 태그(제출 스냅샷)
git tag -a submit-v1 -m "First submission"
git push origin submit-v1
git tag                      # 태그 목록

6) 원격(URL, 권한 관련)
git remote set-url origin https://github.com/<id>/<repo>.git   # 원격 URL 변경
git remote -v                                                  # 확인

7) 되돌리기 & 복구(안전형)
git restore <파일>                      # 워킹디렉토리 변경 취소
git restore --staged <파일>             # 스테이징 취소
git checkout -- <파일>                  # (구버전 방식) 파일 복구

git log --oneline -n 10                # 최근 커밋 요약
git diff                                # 현재 변경 diff
git show <커밋ID>:path/to/file          # 예전 커밋의 특정 파일 보기

8) 이미 올라간 불필요 파일 정리

.gitignore에 규칙 추가한 뒤 이미 추적 중인 것 제거

git rm -r --cached <file_name>
git commit -m "Remove ignored files from repo"
git push


특정 하위 경로일 경우: git rm -r --cached recfun/.idea 처럼 경로 지정.

9) 빈 폴더 올리고 싶을 때(placeholder)
mkdir Assignment1
echo "" > Assignment1\.gitkeep
git add -A
git commit -m "Add Assignment1 skeleton"
git push

자주 나는 에러 & 빠른 진단

푸시했는데 GitHub에 변화 없음 → git add -A 후 git commit을 빼먹었는지 확인

error: src refspec main does not match any → 아직 커밋 없음 → git add -A && git commit -m "msg"

fatal: 'origin' does not appear to be a git repository → git remote add origin <URL> 필요

403 권한 에러 → 원격 URL 계정/토큰 확인, Windows 자격 증명의 github.com 항목 제거 후 재인증

.idea/ target/ 계속 보임 → git rm -r --cached <경로> 후 커밋/푸시
